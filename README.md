# График ВИП — веб-приложение

Форма добавления смен + таблица графика + сводка «ИТОГ по дням» (сотрудники по
остановкам, в алфавитном порядке), с общей базой данных для команды до 10
человек. Работает с любого устройства по одной ссылке.

Ничего программировать не нужно — только пройти шаги ниже. Займёт 10–15 минут.

## Шаг 1. Создать базу данных (Supabase)

1. Зайдите на https://supabase.com и зарегистрируйтесь (можно через Google).
2. Нажмите **New project**. Укажите любое имя, придумайте пароль базы данных
   (сохраните его куда-нибудь) и выберите ближайший регион. Подождите 1–2
   минуты, пока проект создастся.
3. Слева откройте **SQL Editor** → **New query**.
4. Откройте файл `supabase/schema.sql` из этого комплекта, скопируйте всё
   содержимое, вставьте в редактор и нажмите **Run**. Это создаст таблицы
   `employees` и `shifts`.
5. (Необязательно) Если хотите сразу перенести данные недели 15–21.06 из
   Excel-файла — так же откройте **New query**, вставьте содержимое
   `supabase/seed.sql` и нажмите **Run**.
6. Слева зайдите в **Project Settings → API**. Скопируйте:
   - **Project URL**
   - **anon public** ключ (это открытый ключ, не секретный service key —
     именно его нужно использовать в приложении)

## Шаг 2. Выложить код на GitHub

1. Зайдите на https://github.com и создайте аккаунт, если его нет.
2. Создайте новый репозиторий (**New repository**), например `grafik-vip`.
3. Загрузите в него все файлы из этой папки (`grafik-app`) — на странице
   репозитория есть кнопка **uploading an existing file**, можно просто
   перетащить всю папку.

## Шаг 3. Развернуть сайт (Vercel)

1. Зайдите на https://vercel.com и зарегистрируйтесь через GitHub.
2. Нажмите **Add New → Project**, выберите репозиторий `grafik-vip`.
3. В разделе **Environment Variables** добавьте две переменные:
   - `NEXT_PUBLIC_SUPABASE_URL` — вставьте Project URL из шага 1
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` — вставьте anon public ключ из шага 1
4. Нажмите **Deploy**. Через 1–2 минуты Vercel выдаст ссылку вида
   `https://grafik-vip.vercel.app` — это и есть ваше приложение.
5. Разошлите эту ссылку команде — она работает одинаково на телефоне,
   планшете и компьютере, данные общие и сохраняются постоянно.

## Как дальше вносить изменения

Каждый раз, когда меняете код на GitHub, Vercel автоматически пересобирает и
обновляет сайт — вручную ничего перезапускать не нужно.

## Важно про доступ

Ссылка на приложение не защищена паролем — доступ есть у всех, кто её
получит. Это ок для внутренней команды по закрытой ссылке. Если нужна защита
логином/паролем — это можно добавить отдельно (например, через Supabase Auth
или Vercel Password Protection на платном плане), скажите — соберу и это.

## Локальный запуск (для разработчика, необязательно)

```bash
npm install
cp .env.local.example .env.local   # и вписать туда свои ключи
npm run dev
```
